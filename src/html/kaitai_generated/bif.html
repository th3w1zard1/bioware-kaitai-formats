Bif format specification

Type: Bif

**BIF** (binary index file): Aurora archive of `(resource_id, type, offset, size)` rows; **ResRef** strings live in the paired **KEY** (`KEY.ksy`), not in the BIF.

OffsetSizeIDTypeNote
0 ... file_type str(ASCII) File type signature. Must be "BIFF" for BIF files.
4 ... version str(ASCII) File format version. Typically "V1 " or "V1.1".
8 ... var_res_count u4le Number of variable-size resources in this file.
12 ... fixed_res_count u4le Number of fixed-size resources (always 0 in KotOR, legacy from NWN).
16 ... var_table_offset u4le Byte offset to the variable resource table from the beginning of the file.

Parse instance: var_resource_table

var_table_offset ... var_resource_table VarResourceTable Variable resource table containing entries for each resource.

Type: VarResourceEntry

OffsetSizeIDTypeNote
0 ... resource_id u4le Resource ID (matches KEY file entry). Encodes BIF index (bits 31-20) and resource index (bits 19-0). Formula: resource_id = (bif_index << 20) | resource_index
4 ... offset u4le Byte offset to resource data in file (absolute file offset).
8 ... file_size u4le Uncompressed size of resource data in bytes.
12 ... resource_type u4le→BiowareTypeIds::XoreosFileTypeId Aurora resource type id (`u4` on disk). Payloads are not embedded here; KotOR tools may read beyond `file_size` for some types (e.g. WOK/BWM). Canonical enum: `formats/Common/bioware_type_ids.ksy` → `xoreos_file_type_id`.

Type: VarResourceTable

OffsetSizeIDTypeNote
0 ... entries VarResourceEntry Array of variable resource entries.
