Tpc format specification

Type: Tpc

**TPC** (KotOR native texture): 128-byte header (`pixel_encoding` etc. via `bioware_common`) + opaque tail (mips / cube faces / optional **TXI** suffix). Per-mip byte sizes are format-specific — see PyKotor `io_tpc.py` (`meta.xref`).

OffsetSizeIDTypeNote
0 ... header TpcHeader TPC file header (128 bytes total)
128 ... body Remaining file bytes after the header (texture data for all layers/mipmaps, then optional TXI).

Type: TpcHeader

OffsetSizeIDTypeNote
0 ... data_size u4le Total compressed payload size. If non-zero, texture is compressed (DXT). If zero, texture is uncompressed and size is derived from format/width/height.
4 ... alpha_test f4le Float threshold used by punch-through rendering. Commonly 0.0 or 0.5.
8 ... width u2le Texture width in pixels (uint16). Must be power-of-two for compressed formats.
10 ... height u2le Texture height in pixels (uint16). For cube maps, this is 6x the face width. Must be power-of-two for compressed formats.
12 ... pixel_encoding u1→BiowareCommon::BiowareTpcPixelFormatId Pixel encoding byte (`u1`). Canonical values: `formats/Common/bioware_common.ksy` → `bioware_tpc_pixel_format_id` (PyKotor wiki TPC header; xoreos `tpc.cpp` `readHeader`).
13 ... mipmap_count u1 Number of mip levels per layer (minimum 1). Each mip level is half the size of the previous level.
14 ... reserved u1 Reserved/padding bytes (0x72 = 114 bytes). KotOR stores platform hints here but all implementations skip them.
